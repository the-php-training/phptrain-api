## ğŸš€ Key Architectural Patterns

### 1. Domain-Driven Design (DDD)
- **Strategic**: Bounded Contexts, Shared Kernel, Context Mapping
- **Tactical**: Entities, Value Objects, Aggregates, Repositories, Domain Events

### 2. Hexagonal Architecture (Ports & Adapters)
- Domain at center, infrastructure at edges
- Dependency inversion: Domain defines interfaces, Infrastructure implements

### 3. CQRS (Command Query Responsibility Segregation)
- Separate handlers for commands (write) and queries (read)
- Commands: CreateTenantHandler, EnrollStudentHandler
- Queries: GetTenantHandler, ListTenantsHandler

### 4. Event-Driven Architecture
- Domain events for cross-aggregate/context communication
- Synchronous event processing (transactional consistency)
- PSR-14 event dispatcher

### 5. Repository Pattern
- Abstract persistence from domain
- Interface in Domain, Implementation in Infrastructure
- Translation layer: `toDomain()` methods

### 6. Factory Pattern
- `create()` - Create new aggregate
- `reconstitute()` - Rebuild aggregate from persistence

### 7. DTO Pattern
- Data transfer between layers
- Validation at boundaries
- `fromArray()`, `toArray()`, `fromEntity()`
